# UI/UX 개선 리포트

**작성일:** 2025-12-01  
**대상 앱:** Todo Sticker (게이미피케이션 할 일 관리)  
**목적:** 사용자 불편 사항 분석 및 개선 방안 제시

---

## 🔴 주요 문제점 요약

### 1. **기능 과부하 & 정보 밀도 문제**
현재 앱은 너무 많은 기능을 한 화면에 표시하려고 시도합니다:
- 게이미피케이션 (레벨, XP, 스트릭, 성취도)
- 실용성 기능 (카테고리, 태그, 우선순위, 마감일, 하위 할 일)
- 기본 할 일 기능 (추가, 완료, 삭제, 핀)
- 부가 기능 (뽀모도로, 백업, 검색, 정렬)

**결과:** 사용자가 "무엇을 먼저 봐야 하는지", "어디를 클릭해야 하는지" 혼란스러움

---

## 📋 세부 문제 분석

### A. 할 일 입력 영역 (Todo Input)

#### 문제점:
1. **이모지 선택 버튼의 목적 불명확**
   - 위치: `web/index.html:34` (이모지 버튼)
   - 사용자가 "왜 이모지를 선택해야 하는지" 불명확
   - 선택된 이모지가 어디에 표시되는지 즉시 보이지 않음

2. **카테고리/태그/우선순위 설정이 숨겨져 있음**
   - 할 일을 추가한 **후**에만 설정 가능
   - 입력 시점에 함께 설정할 수 없어 두 번 작업 필요

3. **입력 필드가 시각적으로 약함**
   - `web/styles.css:210-240` 참조
   - 배경이 흐릿해서 포커스가 명확하지 않음

#### 개선안:
```
[ 인라인 입력 확장 ]
┌─────────────────────────────────────────┐
│ 📝 할 일 입력...                        │
├─────────────────────────────────────────┤
│ 🏷️ 카테고리 ▼  | 🔖 태그 ▼  | ⭐ 우선순위 ▼│
└─────────────────────────────────────────┘
```
- 입력 필드를 확장 가능한 패널로 변경
- 자주 사용하는 메타데이터를 입력 시점에 함께 설정
- 이모지는 선택 사항이므로 덜 강조

---

### B. 할 일 목록 (Todo List)

#### 문제점:
1. **실용성 메타데이터가 시각적으로 약함**
   - 위치: `web/practicality-manager.js:264-320`
   - 마감일, 우선순위, 하위 할 일이 작은 텍스트로만 표시
   - 긴급한 할 일을 한눈에 파악하기 어려움

2. **할 일 아이템이 너무 복잡함**
   - 체크박스, 이모지, 텍스트, 핀, 뽀모도로, 수정, 삭제 버튼
   - 실용성 메타 (카테고리, 태그, 우선순위, 마감일)
   - **총 10개 이상의 시각 요소**가 한 줄에 배치

3. **하위 할 일 표시가 불명확**
   - `web/styles.css:400-450` (todo-item 스타일)
   - 들여쓰기만으로 계층 표현
   - 부모-자식 관계가 시각적으로 약함

#### 개선안:
```
[ 시각적 우선순위 강조 ]
┌────────────────────────────────────────────┐
│ ⬜ 📝 프로젝트 제안서 작성                  │
│     🔴 높음 | 📅 오늘 | 🏷️ 업무            │
│     └─ ⬜ 자료 조사                         │
│     └─ ⬜ 초안 작성                         │
├────────────────────────────────────────────┤
│ ⬜ 🛒 장보기                                │
│     🟡 보통 | 🏷️ 개인                       │
└────────────────────────────────────────────┘
```
- 우선순위를 색상 배지로 강조 (빨강/노랑/회색)
- 마감일을 "오늘/내일/이번 주" 형태로 단순화
- 하위 할 일을 접을 수 있는 트리 구조로 표시

---

### C. 필터 & 검색 (Filter Bar)

#### 문제점:
1. **필터 상태가 명확하지 않음**
   - 위치: `web/main.js:1654-1710`
   - 필터가 활성화되었는지 한눈에 보기 어려움
   - 어떤 필터가 적용 중인지 확인하려면 클릭 필요

2. **필터 해제가 번거로움**
   - "모두 보기" 버튼이 있지만 시각적으로 약함
   - 각 필터를 개별 해제할 수 없음

3. **검색과 필터가 분리되어 있음**
   - 검색 기능이 백업 패널 안에 숨겨져 있음

#### 개선안:
```
[ 통합 필터 바 ]
┌─────────────────────────────────────────────┐
│ 🔍 검색...  |  🏷️ 카테고리 ▼  |  ⭐ 우선순위 ▼│
├─────────────────────────────────────────────┤
│ 활성 필터: [업무 ✕] [높음 ✕]  전체 해제      │
└─────────────────────────────────────────────┘
```
- 검색, 카테고리, 태그, 우선순위를 한 줄에 통합
- 활성 필터를 배지로 명확히 표시
- 각 배지에 ✕ 버튼으로 개별 해제 가능

---

### D. 게이미피케이션 요소 (Profile Bar)

#### 문제점:
1. **항상 노출되어 공간 차지**
   - 위치: `web/index.html:19-24`
   - 레벨, XP, 스트릭이 상단에 고정
   - 할 일 관리가 주 목적인데 게임 요소가 더 강조됨

2. **성취도 시스템이 너무 복잡함**
   - 28개의 성취도 배지 (`web/constants.js`)
   - 사용자가 "무엇을 해야 배지를 얻는지" 불명확

#### 개선안:
```
[ 접을 수 있는 프로필 ]
┌─────────────────────────────────────┐
│ Lv.5 김철수 ▼  | 🔥 7일 스트릭       │
└─────────────────────────────────────┘

(클릭 시 확장)
┌─────────────────────────────────────┐
│ Lv.5 김철수 ▲                        │
│ ▓▓▓▓▓▓▓▓░░ 320/500 XP               │
│ 🏆 성취도: 5/28                      │
│ 🔥 현재 스트릭: 7일                  │
└─────────────────────────────────────┘
```
- 기본 상태에서는 레벨과 스트릭만 표시
- 클릭하면 상세 정보 펼침
- 게이미피케이션을 원하지 않는 사용자를 위해 "설정에서 숨기기" 옵션 제공

---

### E. 설정 패널 (Settings)

#### 문제점:
1. **너무 많은 설정 항목**
   - 위치: `web/index.html:48-74`
   - 사운드, 효과, 테마, 뽀모도로, 성취도 표시 등
   - 사용자가 "무엇을 켜야 하는지" 혼란스러움

2. **모달 방식의 불편함**
   - 설정을 변경하려면 모달 열기 → 변경 → 닫기 필요
   - 실시간 미리보기 없음

#### 개선안:
```
[ 간소화된 설정 ]
기본 탭:
- 사운드 켜기/끄기
- 다크 모드
- 게이미피케이션 표시

고급 탭:
- 뽀모도로 시간 설정
- 성취도 알림
- 백업/복원
```
- 자주 사용하는 설정만 기본 탭에 표시
- 실시간 미리보기 제공 (예: 사운드 켜면 즉시 테스트음 재생)

---

### F. 모바일 & 터치 반응성

#### 문제점:
1. **버튼 크기가 작음**
   - 수정/삭제 버튼이 작아서 터치하기 어려움
   - 최소 터치 타겟 권장 크기: 44x44px (Apple HIG)
   - 현재: 대부분 32x32px 이하

2. **스와이프 제스처 없음**
   - 삭제/완료를 위해 작은 버튼 클릭 필요
   - 모바일 앱처럼 스와이프로 액션 노출하면 더 편리

#### 개선안:
```
[ 스와이프 액션 ]
왼쪽 스와이프:
├─ 🗑️ 삭제
└─ ⏰ 뽀모도로

오른쪽 스와이프:
├─ ✅ 완료
└─ 📌 핀
```
- 터치 타겟을 최소 44x44px로 확대
- 스와이프 제스처 추가
- 버튼 레이블을 아이콘으로 교체하여 공간 절약

## 🎯 우선순위별 개선 계획

### P0 (즉시 개선 필요)
1. **할 일 입력 간소화**
   - 이모지 선택을 선택 사항으로 변경
   - 카테고리/태그를 입력 시점에 설정 가능하도록

2. **필터 상태 가시성 개선**
   - 활성 필터를 배지로 명확히 표시
   - 개별 해제 버튼 추가

3. **우선순위/마감일 시각화**
   - 색상 배지로 긴급도 표시
   - "오늘", "내일", "이번 주" 형태로 단순화

### P1 (중요도 높음)
4. **게이미피케이션 요소 접기**
   - 프로필 바를 접을 수 있도록
   - 설정에서 완전히 숨기기 옵션 제공

5. **하위 할 일 UI 개선**
   - 트리 구조로 접기/펼치기 가능하도록
   - 부모-자식 관계 시각적 강화

6. **터치 타겟 확대**
   - 모든 버튼을 44x44px 이상으로

### P2 (개선 권장)
7. **스와이프 제스처 추가**
   - 삭제/완료/핀을 스와이프로

8. **설정 패널 간소화**
   - 자주 사용하는 설정만 기본 노출

9. **검색 통합**
   - 백업 패널에서 메인 화면으로 이동
